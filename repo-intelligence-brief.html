How it works

The core: mirror → digest → query
1

Your repos

One list of every repository you want to track — the single source of truth.

2

Daily sync

A scheduled script clones & pulls them all every morning. Always-current local mirror.

3

Weekly digest

Reads the week's merged PRs & releases, writes a plain-English report of what shipped.

4

Ask the code

Anyone asks a question; the assistant reads the source and answers with citations.

Read-only, alwaysThe mirror only ever reads. It never pushes or edits your repos.
Liveness from code"Is X live?" is answered from what actually runs — not from stale docs.
Automate, then verifySync & digest run themselves; a human reads and asks.

The payoff

What actually happens when you ask
You ask
"Do we support webhook retries, and where's that handled?"
Assistant reads the source
Routes to the right repo, opens the real files — guided by liveness rules, no guessing.
You get a cited answer
"Yes — re-enqueued with backoff, max 5 attempts. See api/src/queue/webhook.ts:42."

Why it's trustworthy: the answer comes from the code that actually runs in production — never from a .env.example, a commented-out block, or a doc that says "planned."
